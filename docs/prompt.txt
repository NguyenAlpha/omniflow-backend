ĐÁNH GIÁ THEO 5 NHÓM — mỗi nhóm chấm 0–10:

M1. NOT NULL & default value
Cột nào là NOT NULL mà không có DEFAULT? Thêm cột NOT NULL không DEFAULT vào bảng lớn sẽ lock toàn bộ bảng trên PostgreSQL cũ. Liệt kê các cột nguy hiểm và gợi ý pattern thêm an toàn (add nullable → backfill → set NOT NULL).

M2. Kiểu dữ liệu & resize risk
Cột nào có kiểu dữ liệu dễ phải đổi sau (VARCHAR quá ngắn, INT thay vì BIGINT cho PK, FLOAT thay vì NUMERIC cho tiền tệ)? Đổi kiểu dữ liệu trên bảng lớn cần rewrite toàn bộ — đánh dấu những điểm này.

M3. FK & constraint lock risk
Thêm FK constraint trên bảng lớn sẽ scan toàn bộ bảng để validate. Có FK nào chưa có index đi kèm không? Gợi ý dùng NOT VALID + VALIDATE CONSTRAINT tách biệt để tránh lock.

M4. Rename & drop safety
Bảng/cột nào có tên dễ bị rename sau (quá generic, sai domain)? Rename column trong PostgreSQL an toàn nhưng break JPA entity ngay lập tức nếu không dùng @Column(name=...). Có cột/bảng nào trông như "sẽ bị drop" không — và drop có cascade risk không?

M5. Zero-downtime migration readiness
Schema có hỗ trợ expand/contract pattern không? Tức là: thêm cột mới song song cột cũ → migrate data → xóa cột cũ, mà app vẫn chạy suốt quá trình. Có bảng nào coupling quá chặt khiến pattern này không áp dụng được không?

## 🧠 Kiến trúc mục tiêu

Flow xử lý request:

Request → JWT → SecurityContext
               ↓
         StoreAccessEvaluator

        hit       miss
         ↓         ↓
       return     DB query → save Redis → return
